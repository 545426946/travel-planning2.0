# 首页"立即探索"按钮跳转修改

## 🎯 修改内容

将首页"立即探索"按钮的跳转目标从"热门景点"页面改为"行程规划"页面。

## 📱 修改详情

### 修改前
```javascript
// 跳转到热门景点页面 (Tab 2)
exploreDestinations() {
  console.log('探索目的地');
  this.setData({
    currentTab: 2  // 热门景点页面
  });
}
```

### 修改后
```javascript
// 跳转到行程规划页面 (Tab 1)
exploreDestinations() {
  console.log('探索目的地');
  this.setData({
    currentTab: 1  // 行程规划页面
  });
}
```

## 🗺️ Tab索引说明

| Tab索引 | 页面名称 | 用途 |
|---------|----------|------|
| 0 | 首页 | 首页内容展示 |
| 1 | **行程规划** | ✅ 新的跳转目标 |
| 2 | 热门景点 | 原来的跳转目标 |
| 3 | 我的 | 个人中心页面 |

## 🎨 用户界面影响

### 导航逻辑
- **"立即探索"按钮** → 现在跳转到行程规划页面

- **地图区域** → 跳转到地图页面

1. **更直接的用户路径**: 从首页直接进入行程规划功能

3. **操作流程优化**: 探索 → 规划 → 创建行程的完整流程
